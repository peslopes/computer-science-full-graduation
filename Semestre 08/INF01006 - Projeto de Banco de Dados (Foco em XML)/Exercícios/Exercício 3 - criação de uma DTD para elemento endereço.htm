INF01006_2013_2_TurmaU: Exercício 3 - criação de uma DTD para elemento endereço
Ir para o conteúdo principal

Suponha um elemento XML enderecos. Este elemento contém zero ou mais elementos endereco.

A estrutura do elemento endereco é a seguinte:

O elemento sempre começa com a informação referente a rua e número. Esta informação pode aparecer na forma de um elemento chamado ruanocompl, que contém um string com a rua, o número e o complemento. Alternativamente, podem aparecer um elemento rua, seguido de um elemento numero, seguido de um elemento complemento (este é opcional).

Após, pode aparecer (opcional) um elemento bairro. 

Após vem, obrigatoriamente a cidade com seu cep (opcional), em qualquer ordem. Ambos podem estar em um string dentro do elemento cepcidade, ou dentro de dois strings separados, cep e cidade.

Finalmente, vem um número variado de telefones. Cada elemento telefone contém um atributo com o tipo do telefone, que pode ser R (residencial),  P (profissional) e C (celular). O elemento telefone contém opcionalmente um prefixo e obrigatoriamente um numero.

Construa uma DTD para este tipo de documento e teste com várias instâncias.
